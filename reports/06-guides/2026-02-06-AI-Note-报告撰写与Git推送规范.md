## 1. 报告撰写规范

### 1.1 保存路径
所有报告统一保存至：
```
workspace/AI-Note/reports/
```

### 1.2 文件命名规则
| 要素 | 说明 |
|------|------|
| **日期前缀** | `YYYY-MM-DD-`（如：2026-02-06-） |
| **文件名称** | 使用中文，与报告内容相关 |
| **文件后缀** | `.md`（Markdown 格式） |

**示例**：
- ✅ `2026-02-06-智能网页内容读取器-Claude-Code-Skill分析报告.md`
- ❌ `2026-02-06-url-reader-report.md`（不要用英文）
- ❌ `url-reader-report.md`（缺少日期前缀）

### 1.3 文档格式要求
- 使用 Markdown 格式编写
- 文档末尾**只保留时间戳**，不写本地路径信息

**正确示例**：
```markdown
---

*报告生成时间: 2026-02-06 06:59 CST*
```

**错误示例**：
```markdown
---

*报告生成时间: 2026-02-06 06:59 CST*  
*报告路径: /home/claw/.openclaw/workspace/...*  ← 不要写这个
```

## 2. Git 推送规范

### 2.1 仓库信息
| 项目 | 详情 |
|------|------|
| **本地路径** | `workspace/AI-Note/` |
| **远程仓库** | `https://github.com/Tiramisu-MH/AI-notes.git` |
| **分支** | `main` |

### 2.2 推送时机
写完报告后**自动推送**，无需额外确认。

### 2.3 推送流程
```bash
# 1. 进入仓库目录
cd workspace/AI-Note/

# 2. 添加报告文件
git add reports/

# 3. 提交更改（根据实际情况写提交信息）
git commit -m "提交信息"

# 4. 推送到远程仓库
git push origin main
```

### 2.4 提交信息规范
提交信息应简洁明了，说明本次更改内容：

| 场景 | 提交信息示例 |
|------|-------------|
| 新增报告 | `添加 url-reader 仓库分析报告` |
| 修改报告 | `移除报告中的本地路径信息` |
| 批量更新 | `更新多篇技术报告` |

## 3. 完整工作流示例

以分析一个 GitHub 仓库为例：

```
1. 接收任务：分析某仓库
   ↓
2. 获取信息：web_fetch / web_search
   ↓
3. 撰写报告：保存到 workspace/AI-Note/reports/
   文件名：YYYY-MM-DD-中文名称.md
   ↓
4. 自动推送：
   git add reports/
   git commit -m "添加 xxx 分析报告"
   git push origin main
   ↓
5. 完成任务：告知马博士推送成功
```

---

## 4. 注意事项

1. **命名必须用中文**：体现报告内容，不要使用英文缩写
2. **日期格式统一**：`YYYY-MM-DD-`（年-月-日）
3. **自动推送**：写完后立即推送，无需询问确认
4. **路径保密**：报告末尾不要暴露本地文件系统路径
5. **提交信息清晰**：方便后续查阅历史记录
