## 工作流程 & 偏好

### 📄 大内容发布流程（重要！）
**规则**：当生成大量内容（表格、列表、长文档）时：

1. **不要直接在 Telegram 发送** ❌
   - Telegram 显示混乱（表格变一坨）
   - 历史记录难查找
   - 内容可能被截断

2. **创建 GitHub Pages 页面** ✅
   - 写入 `~/.openclaw/workspace/` 的 Markdown 文件
   - `git add` → `git commit` → `git push`
   - 推送到 `Tiramisu-MH/AI-notes` 仓库
   - GitHub Actions 自动部署

3. **Telegram 只发链接** 🔗
   - 格式："新内容已发布：[标题](https://tiramisu-mh.github.io/AI-notes/文件名)"
   - 简短说明内容要点
   - 方便随时查看和分享

**在线仓库地址**：https://tiramisu-mh.github.io/AI-notes/
